## 🧭 만드는 방식

1. **불편 발견** — 내가 겪었거나, 내가 잘 아는 사람들(자영업자·시니어·상담사)이 겪는 문제
2. **PRD 작성** — JTBD로 문제 정의, MoSCoW로 범위 자르기
3. **바이브 코딩** — Claude Code로 구현, 막히면 원인을 이해할 때까지 파기
4. **배포 & 회고** — 작아도 끝까지. 각 저장소 README에 회고를 남깁니다
